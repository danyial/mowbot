## Hardware Validation Approach

**Firmware Validation (Documented):**

Located in `firmware/src/main.cpp` (lines 311-340), a hardware test sequence runs during `setup()`:

```cpp
// DIAGNOSE: Motor-Test-Impuls
// ENTFERNEN wenn Motoren bestaetigt funktionieren!
delay(5000);  // 5s warten — Zeit zum Einschalten der 12V

set_led(COLOR_YELLOW);       // Gelb = Motor-Test
set_motor_left(0.3f);        // 30% vorwaerts
set_motor_right(0.3f);
delay(1000);

stop_motors();
set_led(COLOR_OFF);
delay(500);

set_led(COLOR_YELLOW);
set_motor_left(-0.3f);       // 30% rueckwaerts
set_motor_right(-0.3f);
delay(1000);

stop_motors();
set_led(COLOR_OFF);
delay(500);

// 3x Gruen blinken = Setup fertig
for (int i = 0; i < 3; i++) {
  set_led(COLOR_GREEN);  delay(100);
  set_led(COLOR_OFF);    delay(100);
}
```

**Validation Method:**
- Manual bench test after firmware upload
- Motors spin forward at 30% speed for 1 second
- Motors spin backward at 30% speed for 1 second
- LED blink sequence (3x green) indicates successful setup
- **Developer note in code:** "ENTFERNEN wenn Motoren bestaetigt funktionieren!" (Remove when motors confirmed working)

## Status Indication via LED

**LED State Machine** (lines 82-91 in `main.cpp`):

The WS2812 RGB LED serves as the primary hardware status indicator:

```cpp
#define COLOR_OFF        0x000000
#define COLOR_RED        0x200000  // Warte auf Agent (Wait for agent)
#define COLOR_YELLOW     0x201000  // Agent gefunden / Motor-Test (Agent found / Motor test)
#define COLOR_GREEN      0x002000  // Verbunden, idle (Connected, idle)
#define COLOR_BLUE       0x000020  // cmd_vel empfangen, Motoren aktiv (cmd_vel received, motors active)
#define COLOR_PURPLE     0x100010  // Disconnected
```

**LED Behavior Patterns:**

- **WAITING_AGENT state:** Blink red (1000ms interval) — searching for micro-ROS agent
- **AGENT_AVAILABLE state:** Solid yellow — creating micro-ROS entities
- **AGENT_CONNECTED state:** 
  - Solid blue when motors active (within `CMD_TIMEOUT_MS` = 500ms of last cmd_vel)
  - Blinking green (500ms interval) when idle (no recent commands)
- **AGENT_DISCONNECTED state:** Solid purple

**Verification:** Operator visually inspects LED color to verify:
- Agent connectivity status
- Motor activity
- System readiness for operation

## Bench Test Procedure (Manual)

**Setup Required:**
- Charged 12V Li-Po battery (XT60 connector)
- Raspberry Pi 4 (powered separately or via HAT)
- 2x motor assemblies (JGB37-520 with encoders) connected to `J4` and `J5`
- 2x BTS7960 motor driver modules connected to `J8` and `J9`
- Serial monitor at 115200 baud connected to ESP32-C3

**Test Steps:**

1. **Power on sequence:**
   - Verify 12V input to screw terminal J6
   - Verify 5V rail generated by MINI560 converter
   - Observe ESP32-C3 boot on Serial monitor

2. **Motor test (automatic during setup):**
   - Wait 5 seconds (delay for 12V stabilization)
   - LED turns yellow (COLOR_YELLOW)
   - Motors spin forward at 30% speed for 1 second
   - Motors stop, LED off, 500ms pause
   - Motors spin backward at 30% speed for 1 second
   - Motors stop, LED off, 500ms pause
   - LED blinks green 3x (setup complete)

3. **Agent connection test:**
   - LED blinks red (WAITING_AGENT) until agent pings on micro-ROS UART
   - Once agent responds, LED turns yellow (AGENT_AVAILABLE) while entities created
   - On successful subscription to `cmd_vel` topic, LED turns blue or green (AGENT_CONNECTED)

4. **Manual motor test (after setup):**
   - Use `rostopic pub` to send velocity commands on `/cmd_vel` topic
   - Observe motors respond and LED turn blue (during active motion)
   - Verify encoder interrupts fire (can add Serial output for debug)

5. **Shutdown:**
   - Send zero velocity command
   - Motors stop, LED returns to green (idle) or red (agent lost)

## Firmware Compilation and Upload

**Build Tool:** PlatformIO

**Configuration File:** `firmware/platformio.ini`

```ini
[env:esp32-c3-devkitm-1]
platform = espressif32
board = esp32-c3-devkitm-1
framework = arduino
monitor_speed = 115200
lib_deps =
    https://github.com/micro-ROS/micro_ros_platformio
    adafruit/Adafruit NeoPixel@^1.12.0
board_microros_distro = humble
board_microros_transport = serial
```

**Build and Upload:**
```bash
cd firmware
pio run -e esp32-c3-devkitm-1 --target upload
pio device monitor -e esp32-c3-devkitm-1  # Serial monitor at 115200 baud
```

**Warnings During Compilation:**
- Expected: Arduino API deprecation notices for `ledcSetup()` (older ESP32 core < 3.0)
- Conditional compilation (`#if ESP_ARDUINO_VERSION_MAJOR >= 3`) handles both APIs
