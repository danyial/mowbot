<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WS → browser DOM | untrusted log-line text crosses into DOM via ansi-to-html |
| server response → UI | /api/logs/containers could return malformed JSON on dockerode error |

## STRIDE Threat Register

| ID | Category | Component | Disposition | Mitigation Plan |
|----|----------|-----------|-------------|-----------------|
| T-06-04 | Information Disclosure (XSS) | ansi-to-html render of crafted log line | mitigate | Instantiate `new Convert({ escapeXML: true, stream: true })`. `escapeXML: true` HTML-escapes every input BEFORE emitting color spans, so `<script>` etc. become inert text. `stream: true` preserves palette state across chunks. Task 3 verify greps the source for `escapeXML:\s*true`. |
| T-06-06 | DoS (memory) | unbounded buffer at 100 lines/s | mitigate | Ring buffer capped at 10,000 lines (UI-SPEC §Performance). Buffer lives in `useRef<string[]>` inside LogViewer — never in Zustand (avoid re-render storm). |
| T-06-07 | Spoofing | WS URL hijack via tampered window.location.host | accept | Same-origin WS; attacker who can rewrite location owns the tab. Trusted-LAN per PROJECT.md. |
| T-06-02 | Tampering (behavioural) | frontend must not break server-side single-upgrade-listener invariant | mitigate | Client-only code here; cannot add server handlers. Wave 0 regression test continues to run in every verify step. |

**Severity:** T-06-04 medium (blocking if escapeXML absent), T-06-06 medium, others accepted.
</threat_model>
