    Script behavior (per D-15):
    1. Accept optional DURATION arg (default 60). Validate it's a positive integer.
    2. Fail-fast guard: `docker exec mower-nav ros2 topic list | grep -q '^/odometry/filtered$'` —
       if absent, print "[yaw-drift-test] /odometry/filtered not publishing" and exit 1.
    3. Capture starting yaw: `docker exec mower-nav bash -c 'ros2 topic echo --once /odometry/filtered
       --field pose.pose.orientation --csv'` → parse into x,y,z,w → feed to python3 one-liner inside
       the container (`docker exec mower-nav python3 -c "..."`) using `tf_transformations.euler_from_quaternion`
       to get yaw in radians. Convert to degrees (`yaw * 180 / pi`).
    4. `sleep ${DURATION}`.
    5. Capture ending yaw via the same pipeline.
    6. Compute `delta_deg = abs(end_deg - start_deg)`; handle wrap-around at ±180° by normalizing to
       [-180, 180] before abs.
    7. Print single final line to stdout:
       `Δyaw = X.XX° over ${DURATION}s — PASS` (exit 0) if `delta_deg < 1.0`
       `Δyaw = X.XX° over ${DURATION}s — FAIL (<1°)` (exit 1) otherwise.
    8. Chmod +x the script (`chmod +x scripts/yaw-drift-test.sh`).

    Implementation note: all quaternion math happens inside the nav container (tf_transformations is
    pre-installed there per RESEARCH §Standard Stack). The outer bash stays POSIX-simple.

    Do NOT write output to VERIFICATION.md automatically (D-17: manual paste). Do NOT make the script
    depend on anything in config/ekf.yaml — it reads `/odometry/filtered` which exists whether or not
    `pose0` is wired (Wave 1's job). Running this script BEFORE Wave 1 edits should still succeed
    against the current IMU-only EKF (likely FAIL the <1° criterion — that's fine; it proves the script
